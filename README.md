# MoodRun Process Portfolio

MoodRun is the CPT208 Human-Centric Computing group project for **C1 Go Runners**. This repository contains the **Process Portfolio** only. It documents the design process behind MoodRun, an emotion-aware running companion that connects mood check-ins, playful route choice, adaptive running support, and post-run reflection.

The functional web app system is maintained separately and is linked below.

## What This Portfolio Covers

- A 200-word motivation statement explaining why the group selected the Active Lifestyles track.
- Academic and commercial research comparing what existing solutions do well and what they miss.
- Primary and secondary stakeholder definition with personas.
- User journey mapping, pain points, and playful system requirements.
- Evidence of user research and prototype interaction.
- Crazy Eights sketches and alternative design directions.
- Low-fi prototype link and rationale for the selected concept.
- System architecture diagram and high-fi prototype links.
- Individual contribution table for all four members.
- Usability testing results from the alpha prototype.
- Before-and-after refinement evidence based on user feedback.
- Final reflection on social, ethical, accessibility, privacy, and AI-use considerations.

## Local Preview

Open `index.html` directly in a browser, or run a simple local static server from the repository root:

```bash
python -m http.server 8000
```

Then visit:

```text
http://localhost:8000
```
