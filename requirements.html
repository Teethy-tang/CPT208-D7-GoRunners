2. User Requirements

Structured by coursework requirements: User Journey Map, Requirements List, Evidence of Life.

User Journey Map (Current Experience)

Pain Points Summary

  • No simple way to convert current mood into an achievable running plan.
  • Most feedback systems prioritize performance over emotional readiness.
  • Post-run data does not help users understand motivation fluctuations.

Requirements List

Three must-haves for the system to be considered playful.

Requirements List: Playful Must-Haves

  • The system must transform mood check-ins into personalised running experiences, so users feel that each run has a playful emotional theme rather than being a generic fitness task.
  • The system must provide game-like feedback during and after running, including progress indicators, encouraging cues, mood-based visuals, music, and a before/after mood reflection.
  • The system must support self-expression and low-pressure exploration through features such as avatar customisation, route choices, random route generation, and free-running guidance that encourages play instead of punishment.
