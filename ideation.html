Design Alternatives

Compare interaction models and justify the selected direction.

MoodRun had several design conflicts because the app needs to support both playful freedom and meaningful running guidance. We compared different interaction models before choosing the final approach.

1. How Users Express Emotions

Alternative Description Limitation
Text Input Only Users freely type how they feel before running. Emotion expression is rich, but it takes more effort and may slow down the flow.
Mood Selection Only Users choose from preset mood options. Fast and simple, but emotional expression can feel limited.
Hybrid Mood Check-in Users can select a mood and optionally write a short reflection. A local rule-based Mood AI can suggest the closest mood. Slightly more complex, but gives both speed and personalisation.

Chosen Approach: Hybrid Mood Check-in

We chose the hybrid approach because it best supports MoodRun's human-centred playful design. Users who want a quick start can simply choose a mood, while users who want deeper reflection can write their thoughts. The local mood assistant keeps the experience personal without needing an external AI service.

2. How to Handle Plan Distance vs User Route Distance

Alternative Description Limitation
Force Route to Match Plan The system automatically creates a route that exactly matches the plan distance. Difficult to implement reliably in a front-end prototype; road safety, walkability, rivers, bridges, and route quality are hard to guarantee.
Ignore the Difference The app lets users choose any route without explaining the distance mismatch. Simple, but users may feel confused when the route distance and plan target do not match.
Show Distance Difference and Let Users Decide The app displays the plan distance, route distance, and difference. If the difference is large, users choose either "Use Route Distance" or "Keep Plan Target". Requires an extra decision, but it is transparent and flexible.

Chosen Approach: Distance Difference + User Choice

We chose to show the distance conflict clearly. For example, if the plan is 4.0 km but the selected route is 2.3 km, MoodRun can show the difference and let the user decide whether to use the actual route distance or keep the original plan target. This fits MoodRun better than forcing an automatic detour, because the app is a playful running companion, not a strict navigation system.

3. How Strictly Users Must Follow the Route

Alternative Description Limitation
Strict Route Completion The run only counts if the user follows the recommended route closely. Too restrictive and makes the app feel like a serious navigation tool.
Free Running Only The route is ignored once the run starts. Gives freedom, but the route setup feature becomes less meaningful.
Recommended Route with Gentle Guidance The route is treated as a recommendation. The app gives start-point and off-route prompts, but does not stop or punish the user. Less strict, but better matches playful outdoor movement.

Chosen Approach: Recommended Route with Gentle Guidance

We chose to treat routes as recommendations. If the user starts far from the planned start point, MoodRun can display a gentle warning. If the user moves away from the route, the app can show an off-route message. However, the run continues normally, and the final summary records the user's real GPS distance, time, pace, and mood change. This keeps the experience flexible, supportive, and playful.
